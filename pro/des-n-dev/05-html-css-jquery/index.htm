С появлением элементов стуктуры документ можно делить на секции двумя способами: «неочевидные» секции разделяются между собой заголовками; очевидные заключаются в элементы структуры. При этом неочевидные секции можно создавать внутри очевидных. А очевидные внутри неочевидных нет.

То есть следующий фрагмент написан правильно: в очевидной секции — две неочевидных.

<section>
<h2>Mares</h2>

<h3>Pink Diva</h3>
<p>Pink Diva has given birth to three Grand National winners.</p>

<h3>Ring a Rosies</h3>
<p>Ring a Rosies has won the Derby three times.</p>
</section>

А вот этот фрагмент написан неправильно: две очевидных секции не становятся разделами неочевидной секции — они будут «соседями».

<h2>Mares</h2>
<article>
<h3>Pink Diva</h3>
<p>Pink Diva has given birth to three Grand National winners.</p>
</article>
<article>
<h3>Ring a Rosies</h3>
<p>Ring a Rosies has won the Derby three times.</p>
</article>

Исключения составляют «корневые неочевидные» секции, созданные элементами body, td, blockquote, details, fieldset, figure. То есть следующий фрагмент логически верен:

<body>
<h1>Mares</h1>
<article>
<h2>Pink Diva</h2>
<p>Pink Diva has given birth to three Grand National winners.</p>
</article>
<article>
<h2>Ring a Rosies</h2>
<p>Ring a Rosies has won the Derby three times.</p>
</article>
</body>

Причем если первый заголовк в документе будет заключен в элемент структуры, заголовком документа будет считаться следующий за ним заголовок

<section>
<h1>Этот H1 не будет заголовком документа</h1>
</section>
<h6>Заголовком документа будет этот H6</h6>

Здесь возникает одна проблема. Если отдельную заметку на странице помещать в article весь документ остается безымянным. Некоторые считают, что первый уровень структуры документа нужно размечать с помощью div, а article, section, aside и nav использовать только для деления на разделы. Я пока не использовал такой подход. Но, по0моему, он логичен.

Правило заголовка. Если блок документа нельзя озаглавить, то следует использовать div, а не article или section. Хотя последние два элемента в принципе могут быть не озоглавлены (например, в случае комментариев), чаще всего элементы структуры нуждаются в заголовках, элементы группировки нет.

<article>
Самостоятельный блок содержания — заметка блога, анонс в ленте, комментарий в форуме или блоге. <article> может включать другие элементы <article>: как, например, статья <article> и комментарии к ней <article>
<aside>
This is usually expressed in sidebars that contain elements like related posts, tag clouds, etc. They can also be used for pull quotes. Can contain nav elements.
<audio>
Defines sound content
<canvas>
Defines graphics
<command>
defines a command button, like a radiobutton, a checkbox, or a button.
<datalist>
defines a list of options (like <select> previously). Use this element together with the input element, to define which values the input element can have.
<details>
Используется не только для подробностей документа или какой-то его части, но и для экспандо бокса — выдвигающейся панели действий.
<figure>
Defines a group of media content, and their caption (optional elements <figcaption>). It must be inside a menu element.
<figcaption>
Defines the caption of a figure element
<embed>
defines embedded content, such as a plug-in.
<header>
The header element contains introductory information to a section or page.
It can also be used to wrap a section's table of contents, a search form, or any relevant logos. Может содержать <nav>
<hgroup>
На мой взгляд, при наличии header, hgroup не нужен.
<keygen>
Defines a generated key in a form
<mark>
как <em>, <strong>, <b>
<meter>
defines a measurement. Used only for measurements with a known minimum and maximum value. Displaying a scalar measurement within a known range, like hard drive usage, can be done using the meter element (spec: 4.10.17)
<nav>
The nav element is reserved for a section of a document that contains links to other pages or links to sections of the same page. Not all link groups need to be contained within it, just primary navigation. Can contain headings.
<progress>
Шкала выполнения задачи. can be both indeterminate as determinate.
<section>
<section> — в отличие от div-а, выполняющего в основном оформительские функции, семантическая часть документа. <section> может быть не только контейнером для содержания, но и контейнером для частей содержания. Так, <section> может включать несколько <article>. Но и <article> может содержать несколько <section>.
<small>
Используется как семантический элемент для выделения сносок, комментариев.
<time>
<time datetime="1975-10-12">День рождения моей любимой женщины</time>
<footer><time datetime="2011-10-12">21 ноября</time> Владимир Никишин, Esquier</footer>
