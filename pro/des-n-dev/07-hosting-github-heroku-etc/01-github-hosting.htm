GitHub / GitHub Pages
для хостинга

Ограничения и предостережения

  1. К сожалению, GitHub Pages не поддерживает технолгию SSI — повторяющиеся фрагменты кода приходится держать в каждом файле. Альтернатива — перейти на Jekyll.
  2. GitHub Pages игнорирует внутренние ссылки, фрагменты которых начинаются с нижнего подчеркивания — их следует избегать. Или, если не используется Jekyll, добавить в корневой каталог пустой файл .nojekyll.
  3. Не следует чистить Mac OS от системных файлов ничем, кроме Onyx и CleanMyMac — это может привести к сбоям в работе репозиториев и, как следствие, сайтов.
  4. Сбои случаются. В этом случае надо писать в службу поддержки — реагируют быстро.

Публикация сайта на хостинг GitHub Pages

  1. Создать репозиторий.
    • Либо — в GitHub.com; флаг на Initialize this repository with a README. После создания на странице репозитория нажать кнопку Clone in Mac.
    • Либо добавить в папку с готовым сайтом файлы .gitignore и README.md (как минимум, название проекта); и перетащить папку в GitHub for Mac. Нажать Push to GitHub в правом верхнем углу. На вопрос «Создать локальный репозиторий?» ответить согласием. Называть репозитории латинскими буквами — GitHub использует эти названия в URL, и кириллица в них превращается в набор дефисов.
  2. В GitHub for Mac, в панели Branches создать ветку gh-pages локального репозитория.
  3. В Mac-GitHub в Settings в Ignored files добавить список файлов, которые не нужно синхронизировать. 
    # Souce Folders
_stock/

# Souce files
*.psd

# Logs and databases
*.log
*.sql
*.sqlite

# OS generated files
.DS_Store
.DS_Store?
._*
.Spotlight-V100
.Trashes
Icon?
ehthumbs.db
Thumbs.db

# Misc
wp-*.php
wp-content/upgrade/*
sitemap.xml
sitemap.xml.gz
  4. Затем сделать первый коммит — GitHub for Mac, панель Changes, кнопка Commit & Sync.
  5. Всё, сайт — в сети. Адрес
    http://username.github.com/repo-name/
    Например:
    http://supervova.github.com/sms-novel/
  6. Если в Dreamweaver-е сайт еще не создан, создать из той же директории .

Уникальное доменное имя

  1. Проверить в GitHub for Mac, что текущая ветка — gh-pages. Создать текстовой файл без расширения CNAME с одной строкой — доменным именем (без www). Например,
    example.com
  2. В личном кабинете компании-регистратора надо открыть страницу настроек нужного домена (например, в Агаве надо щелкнуть на пиктограмму «шестеренка» в колонке «Операции», а затем выбрать «Управление зоной DNS»).
  3. В блоке настроке «Запись типа A» (сокращение от «Адреса») указать IP-адрес github, в который будет транслироваться домен: в левом поле первой строки формы должен быть проставлен знак @ (символ означает домен), а в правом — IP-адрес github (на момент написания — 204.232.175.78).
  4. В блоке «Запись типа CNAME» в левом поле первой строки — www, в правом поле —доменное имя без www, но с точкой на конце (например, example.com.)

Повседневная работа

  1. Изменения, требующие синхронизации, собираются в панели Changes.
  2. История изменений — в GitHub for Mac, в панели History или на GitHub.com, во вкладке Commits проекта.
  3. Командная работа. Утром, перед работой загрузить последнюю версию с GitHub.com.
  4. Вечером после работы переписать измененные файлы в удаленный репозиторий — GitHub for Ma > Changes > Commit & Sync&
  5. Чтобы перейти на сайт из в GitHub for Mac, в панели Settings нажать кнопку View on GitHub.
  6. Для того, чтобы добавить изменения в gh-pages в ветку master, в GitHub for Mac, в панели Branches в правом верхнем углу есть кнопка Merge View .
