Установка и настройка

  1. Если не установлена и не настроена XAMPP, устанавливаем и настраиваем.
    Если требуется новая версия PHP, обновляем XAMPP:
    • Если компьютер не подключен к машине времени, сделать резервную копию xamppfiles/var/mysql, etc/httpd.conf и etc/extra/httpd-vhosts.conf
    • выключить сервер
    • переустановить XAMPP
    • восстановить из машины времени или перезаписать из резервной копии папки сайтов и файлы пользователей из /mysql/, а также httpd.conf и httpd-vhosts.conf
    • включить сервер.
  2. Меняем настройки php.ini
  3. Скачиваем Drupal, распаковываем во временную папку.
  4. Удаляем избыточную информацию и дубликаты:
    • .gitignore
    • .htacess
    • CHANGELOG.txt
    • COPYRIGHT.txt
    • INSTALL.mysql.txt
    • INSTALL.pgsql.txt
    • INSTALL.sqlite.txt
    • MAINTAINERS.txt
    • README.txt
    • robots.txt
  5. Переписываем Drupal в папку сайта.
  6. Копируем .htaccess из templates
  7. sites > default
    Дублируем default.settings.php, переименовываем в settings.php, проверяем свойства — должно быть разрешено редактирование.
  8. В Dreamweaver-e настраиваем сайт. В разделе servers добавляем новый (или редактируем уже созданный) тестовый сервер):

  9. http://localhost/phpmyadmin/
    Create new database > вводим имя бызы, все остальные настройки оставляем неизменными > создаем пустую БД. (На всякий случай, перезапускаем Apache.)
  10. www.new-site.dev/install.php. Появляется мастер установки Drupal.
    На одном из шагов установки потребуется ввести данные БД. Username: root. Password оставляем не заполненным.
    Если в процессе возникают ошибки, см. возможные решения здесь.
  11. > Configuration > Media > File System
    • a/upload
      a/upload/private (Drupal создаст htaccess, запрещающий доступ к этой папке)
    • _tmp (Drupal создаст htaccess, запрещающий доступ к этой папке).
  12. (Management menu; /admin/) > Edit Shortcuts (вторая строка, справа)
    Добавляем пункт Perfomance, путь admin/config/development/performance
    Изменения иногда появлялись только после выхода из системы и повторного входа.
  13. > Appearance > вкладка Settings (справа) > пункт Global settings (справа) > раздел Toggle display. Из всех элементов страницы оставляем только:
    • Site name
    • Site slogan
    • Main menu
    • Secondary menu
    • User pictures in comments (если предполагается использование Gravatar)
  14. Скачать файлы русского языка.
    • Сохраняем в profiles/standard/translations/
    • Убедиться, что включен модуль Locale
    • Для перевода «ядра» Добавить их на странице Translate interface.
    • Для перевода модулей создать в папке нужного модуля папку po и переписать туда соответствующий файл.
    • Записать в «Напоминания» — установить русский языком по умолчанию перед сдачей русскоязычному заказчику.
