Роли, пользователи, права

> People > вкладки List и Permissions (подразделы Permissions и Roles)

  • Для использования любого модуля или возможности нужны права. Чтобы давать или ограничивать права большому количеству пользователей в D предусмотрены роли. По умолчанию это авторизованный и неавторизованный пользователи, администратор сайта.
  • Можно создавать собственные роли: например, «сотрудник» или «редактор».
  • После установки всех модулей и создания всех ролей, надо распределить права.
  • Чтобы отметить сразу все чекбоксы прав, можно в панели FF Web Developer выбрать Forms > Populate Form Fields
  • Если сайт содержит пользовательские страницы, в чьих URL-ах, как правило, используются имена пользователей, нельзя разрешать пользователям менять эти имена.
  1. > People > Permissions > Roles > Add Role
    Создаем роль «Редактор»
  2. Edit permissions. Даем права на:
    • Comment (все действия)
    • Content translation
    • Dashboard
    • Use the Filtered HTML text format
    • Image
    • Blog entry: Create new content
    • Blog entry: Edit own content
    • Blog entry: Edit any content
    • Overlay
    • Set Page Title
    • Create and edit URL aliases
    • View the administration theme
    • View site reports
    • Block IP addresses
    • Use the administration toolbar
  3. > Configuration > Content authoring > Text formats > Filtered HTML > Limit allowed HTML tagsEnabled
    Заменить значение по умолчению на:
    <a> <audio> <blockquote> <cite> <code> <dd> <dl> <dt> <em> <embed> <h3> <img> <li> <object> <param> <strong> <ol> <ul> <video>

Где-то здесь в D6 были еще настройки рассылки писем пользователям (welcome,forgot password, confirmation), но я их пока не нашел. Вполне возможно, что для них нужны Simplenews и Mime Mail.
