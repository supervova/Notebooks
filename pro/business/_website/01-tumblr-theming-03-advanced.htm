Свой шаблон для tumblr-блога. Настройки

Специальные страницы (вроде About)

{block:HasPages} {/block:HasPages} Блок отображается в том случае, если в настройках определены какие-то специальные страницы.
{block:Pages} {/block:Pages} Блок отображается на каждой из специальных страниц.
{URL} URL этой страницы
{Label} Текст ссылки на эту страницу

Подписка / Following

{block:Following} {/block:Following} Отображается, если автор блога подписан на чужие блоги
{block:Followed} {/block:Followed} Отображается для каждого блога, на который подписан пользователь
{FollowedName} Автора блога, на который подписан пользователь
{FollowedTitle} Название блога, на который подписан пользователь
{FollowedURL} URL блога, на который подписан пользователь
{FollowedPortraitURL-16}
[24-30-40-48-64-96-128]		 Портрет фвтора блога, на который подписан пользователь, разных размеров.

Пример

{block:Following}
Blogs I follow:
<ul>
{block:Followed}
<li>

<img src="{FollowedPortraitURL-48}"/>
<a href="{FollowedURL}">{FollowedName}</a>
</li>
{/block:Followed}

</ul>
{/block:Following}

«Лайки»

{block:Likes} {/block:Likes} Отображается, если в настройках «лайки» сделаны публичными.
{Likes} HTML «лаек».
{Likes limit="5"} HTML 5 последних «лаек» (максимум — 10)
{Likes width="200"} HTML «лаек» с аудио и видеоплейерами по 200 пикселей в ширину (ширину картинок нужно указывать в css с помощью свойства max-width)
{Likes summarize="100"} HTML «лаек» с кратким изложением до 100 знаков (максимум — 250)

Пример

...
li.like_post { float: left; padding: 0 40px 0 0; width: 150px
/* ширина должна соответствовать ширине tumblr-тега «лаек» */}
li.like_post IMG { max-width: 100%}
...
{block:Likes}
<div id="likes_container">
<h2>Нам понравилось</h2>
{Likes limit="5" summarize="100" width="150"}
<a href="http://www.tumblr.com/liked/by/{Name}">И всё в том же духе</a>
</div>
{/block:Likes}
...

Twitter

Если в настройках Tumblr включена интеграция с Twitter в tumblr-блог можно включать /tweets.js, который будет показывать последние записи в личной twitter-ленте. Этот скрипт запускает функцию recent_tweets() с единственным параметром — данными Twitter API JSON.

{block:Twitter} {/block:Twitter} Отображается, если в настройках включена интеграция с Twitter
{TwitterUsername} Пользовательское имя автора tumblr-блога в Twitter.

Пример

{block:Twitter}
<div id="twitter" style="display:none"><!--Непонятен смысл display:none — требует проверки экспериментом-->
<h3><a href="http://twitter.com/{TwitterUsername}">Latest Tweets</a></h3>
<div id="tweets"></div>        
</div>
    
<script type="text/javascript">
function recent_tweets(data) {
	for (i=0; i<data.length; i++) {
		document.getElementById("tweets").innerHTML = 
		document.getElementById("tweets").innerHTML + 
		'<a href="http://twitter.com/{TwitterUsername}/status/' + 
		data[i].id + '"><div class="content">' + data[i].text + 
		'</div></a>';
	}
	document.getElementById("twitter").style.display = 'block';
}
</script>
{/block:Twitter}
      
<!-- Put this at the bottom of the page -->
{block:Twitter}
<script type="text/javascript" src="/tweets.js"></script>
{/block:Twitter}

Пользователю можно разрешить менять цвета шаблона

Если включить специальные мета-теги в шаблон, пользователь сможет менять цвета шаблона в разделе Customize.

Пример

...
<!-- DEFAULTS -->
<meta name="color:Background" content="#eee"/>
<meta name="color:Content Background" content="#fff"/>
<meta name="color:Text" content="#000"/>
...
BODY { background: {color:Background}}
#content { background: {color:Content Background}; color: {color:Text}}
...

Пользователю можно разрешить менять шрифты

Если включить специальные мета-теги в шаблон, пользователь сможет менять шрифты в разделе Customize.

Пример

...
<!-- DEFAULTS -->
<meta name="font:Title" content="Helvetica Neue"/>
<meta name="font:Body" content="Arial, Helvetica, sans-serif"/>
...
H1 { font: 30px {font:Title}}
#content { font: 12px {font:Body}}
...

Пользователю можно разрешить скрывать/показывать блоки

Если включить специальные условные мета-теги в шаблон, пользователь сможет скрывать и показывать отдельные блоки или элементы дизайна. Атрибут мета-тега name: if: + в свободной форме описание опции, с пробелами между словами. С помощью атрибута content указывается значение по умолчанию (1— да, 0 — нет). В HTML блоки помещаются в специальные теги {block:If…} и/или {block:IfNot…}, где вместо троеточия указывается описание опции без пробелов.

Пример 

...
<!-- DEFAULTS -->
<meta name="if:Show people I follow" content="1"/>
<meta name="if:Reverse pagination" content="0"/>
...
{block:IfNotReversePagination}
<a href="...">Позже</a> <a href="...">Раньше</a>
{/block:IfNotReversePagination}
...
{block:IfReversePagination}
<a href="...">Раньше</a> <a href="...">Позже</a>
{/block:IfReversePagination}
...
{block:IfShowPeopleIFollow}
<div id="following">...</div>
{/block:IfShowPeopleIFollow}
...

Надписи с изменяемыми фрагментами

Если включить специальные мета-теги в шаблон, пользователь сможет использовать переменные в надписях, включенных в шаблон. Полезно для виджетов с именами пользователей.

Пример 

...
<!-- DEFAULTS -->
<meta name="text:Flickr Username" content=""/>
...
{block:IfFlickrUsername}
<div id="flickr_widget">
<script type="text/javascript" src="http://flickr.com/widget?user={text:Flickr Username}"></script>
</div>
{/block:IfFlickrUsername}
...

Пользователю можно разрешить менять графику

Если включить специальные мета-теги в шаблон, пользователь сможет менять отдельные картинки, включенные в шаблон. Полезно для виджетов с именами пользователей. Если значение переменной изображения (например, {image:Logo}) не определено, в данной области страницы будет добавлен прозрачный 1-пиксельный gif.

Пример

...
<!-- DEFAULTS -->
<meta name="image:Background" content="http://static.tumblr.com/..."/>
<meta name="image:Header" content=""/>
...
BODY { background: #2D567C url("{image:Background}")}
...
{block:IfHeaderImage}<img src="{image:Header}"/>{/block:IfHeaderImage}
{block:IfNotHeaderImage}<h1>{Title}</h1>{/block:IfNotHeaderImage}
...

Пользовательские CSS

С помощью тега {CustomCSS} включенного последней строкой в блок стилей, пользователи смогут менять тему, не редактируя разметку.

Пример

...
<style type="text/css">
#content { background: #FFF; color: #000}
{CustomCSS}
</style>
...

Страницы отдельного дня

Tumblr-блог, может показывать записи за отдельный день http://supervova.tumblr.com/day/2011/03/01/
Используя разметку, можно сделать навигацию типа «Ранее — Позже».

{block:DayPage} {/block:DayPage} Блок отображается только на странице отдельного дня.
{block:DayPagination} {/block:DayPagination} Блок отображается в том случае, если есть предыдущие и/или следущие страницы.
{block:PreviousDayPage} {/block:PreviousDayPage} Блок отображается в том случае, если есть предыдущие страницы отдельных дней — «ранее».
{block:NextDayPage} {/block:NextDayPage} Блок отображается в том случае, если есть следующие страницы отдельных дней — «позже».
{PreviousDayPage} URL предыдущей страницы отдельного дня — «ранее».
{NextDayPage} URL следующей страницы отдельного дня — «ранее».

Пример

...
<h1>{Title}</h1>

{block:DayPage}
	<h2>{Month} {DayOfMonth}, {Year}</h2>
{/block:DayPage}

<ol id="posts">
{block:Posts}
...
{/block:Posts}
</ol>

<div id="footer">
{block:Pagination}
	
{block:PreviousPage}
<a href="{PreviousPage}">&#171; Previous</a>
{/block:PreviousPage}

{block:NextPage}
<a href="{NextPage}">Next &#187;</a>
{/block:NextPage}

{/block:Pagination}

{block:DayPagination}

{block:PreviousDayPage}
<a href="{PreviousDayPage}">&#171; {ShortMonth} {DayOfMonth}</a>
{/block:PreviousDayPage}

{block:NextDayPage}
<a href="{NextDayPage}">{ShortMonth} {DayOfMonth} &#187;</a>
{/block:NextDayPage}

{/block:DayPagination}
</div>
</body>
</html>
