Свой шаблон для tumblr-блога. Базовая техника

Два вида специальных операторов

Переменные. Используются для отображения динамической или пользовательской информации, вводимой в формах настроек.

<title>{Title}</title>

Блоки. Используются для отображения комплектов информации (например, заметок) или html-блоков, появляющихся при определенных условиях (ссылка на предыдущую страницу):

<ol id="posts">
{block:Posts}
<li> ... </li>
{/block:Posts}
</ol>

Пример полного шаблона

Переменные общего назначения

В принципе, тему можно выложить в Theme Garden — как дополнительный промо-материал. Поэтому лучше использовать все переменные.

Навигация

{block:Pagination} {/block:Pagination} Блок отображается в том случае, если есть предыдущие и/или следущие страницы.
{block:PreviousPage} {/block:PreviousPage} Блок отображается в том случае, если есть предыдущие страницы (более свежие записи).
{block:NextPage} {/block:NextPage} Блок отображается в том случае, если есть следующие страницы (более ранние записи).
{PreviousPage} URL предыдущей страницы (более свежие записи).
{NextPage} URL следующей страницы (более ранние записи).

{block:SubmissionsEnabled} {/block:SubmissionsEnabled} Блок отображается в том случае, если в настройках разрешены подписки.
{block:AskEnabled} {/block:AskEnabled} Блок отображается в том случае, если в настройках разрешена форма вопроса.
{AskLabel} Надпись к ссылке/форму вопроса (например, Ask me anything)
{block:PermalinkPagination} {/block:PermalinkPagination} Блок отображается на странице записи для перехода к предыдущим и следующим записям
{block:PreviousPost} {/block:PreviousPost} Блок отображается на странице записи в том случае, если есть страница предыдущей (более свежой) записи — «Более свежее».
{block:NextPost} {/block:NextPost} Блок отображается на странице записи в том случае, если есть страница следующей (более ранней) записи — «Ранее».

{NextPost} URL следующей (более ранней) записи

Общие элементы записей
{Permalink} Ссылка на страницу записи
{ShortURL} Корткий URL страницы записи. (Например: «http://tumblr.com/xpv5qtavm")
{PostID} ID записи. (Например: 1234567)
{TagsAsClasses} Набор СSS-классов записи, составляемый из ее тегов. Пробелы и точки заменяются нижним подчеркиванием. (Например: «humor office new_york_city").

По умолчанию в список классов добавляются классы источника перепечатки ("twitter_com", «digg_com", etc.) и класс reblog, если перепечатка сделана из другого tumblr-блога.
{block:Post[1-15]} {/block:Post[1-15]} Запись на странице отображается в установленном порядке. Полезно для постоянных блоков рекламы или элементов дизайна в ленте.

Например: {block:Post4}я всегда буду отображаться на странице четвертым сверху{/block:Post4}
{block:Odd} {/block:Odd} Блок отображается для каждой нечетной записи в ленте
{block:Even} {/block:Even} Блок отображается для каждой четной записи в ленте
{block:More} {/block:More} Отображается для длинных записей, если при публкации выбрана опция зсокращения записи в ленте до анонса

Теги и страницы тегов

{block:HasTags} {/block:HasTags} Блок отображается внутри {block:Posts}, если запись имеет теги.
{block:Tags} {/block:Tags} Rendered for each of a post's tags.
{Tag} Собственно тег.
{URLSafeTag} «Безопасная» версия URL тега (возможно, «безопасный» означает страницу исключительно данного тега; записи под несколькими тегами сюда уже не попадают)
{TagURL} URL тега. Возможно такая страница, на которой отображаются записи и под одним и под несколькими тегами.
{TagURLChrono} URL тега. На странице тега записи собраны в хронологическом порядке.
{block:TagPage} {/block:TagPage} Блок отображается на странице тега.

Примеры

<html>
<body>
{block:Posts}
<div class="post">
{block:Text}...{/block:Text}
{block:Photo}...{/block:Photo}
...

{block:HasTags}
<ul class="tags">
{block:Tags}
<li>
<a href="{TagURL}">{Tag}</a>
</li>
{/block:Tags}
</ul>
{/block:HasTags}

</div>
{/block:Posts}
</body>
</html>
